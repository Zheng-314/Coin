# ADR-002: 双图输入 + VGG19 架构选择

## 状态

已采纳（2025 年 10 月）

## 背景

PCGS 钱币鉴定需要同时观察正反两面，因此模型需要接收两张图片作为输入。

## 决策

采用 **VGG19 + SE（Squeeze-and-Excitation）双路特征提取 + 拼接分类** 架构。

## 理由

1. **双路输入**：正反面各走一个 backbone，特征图在 FC 层前拼接，保留空间对应关系
2. **VGG19**：ImageNet 预训练权重成熟，对小数据集微调效果好
3. **SE 模块**：在 5 个 maxpooling 层后插入通道注意力，提升特征表达（约 2-3% 准确率）
4. **预训练权重**：ImageNet1K_V1 提供良好的底层特征

## 架构

```
正面(224×224) → VGG19+SE → [512×7×7]
                                    ↘ Concat → FC → 输出
反面(224×224) → VGG19+SE → [512×7×7]
```

## 代价

- 参数量较大（约 140M），不适合移动端部署
- 双路推理耗时约为单路的 2 倍

## 备选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **ResNet50** | 更深的网络，梯度稳定 | 特征图尺寸偏大，拼接后 FC 参数量爆炸 |
| **EfficientNet** | 参数少、效率高 | 预训练权重在钱币领域微调效果不如 VGG |
| **ViT** | 全局注意力 | 需要更大数据集（>100K）才能收敛 |
