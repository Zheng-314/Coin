## 一、知识库概览

本项目知识库由两部分组成：

| 组件 | 存储 | 数量 | 用途 |
|------|------|------|------|
| 结构化钱币数据 | SQLite (`item.db`) | 500 条 | Q&A 关键词检索兜底 |
| 知识图谱 | GraphRAG (Parquet + ChromaDB) | 实体+关系+社区 | 本地/全局语义搜索 |

## 二、item.db 数据统计

### 2.4 数据特点

- **全为 PCGS/NGC 评级币**：记录包含评级信息和品相描述
- **中英混合**：标题和描述均包含中英文信息
- **机制银币为主**：500 条全部属于 `jizhiyinbi` 大类
- **数据量偏少**：仅 500 条，覆盖面有限

## 三、知识图谱数据

### 3.1 GraphRAG 构件

| 文件 | 格式 | 说明 |
|------|------|------|
| `entities.parquet` | Parquet | 知识图谱实体（钱币、人物、地点、事件等） |
| `relationships.parquet` | Parquet | 实体间关系 |
| `text_units.parquet` | Parquet | 原始文本单元 |
| `communities_reports.parquet` | Parquet | 社区摘要报告 |
| `merged-graph.pickle` | Pickle | 合并图 |
| `summarized-graph.pickle` | Pickle | 社区摘要后的图谱 |

### 3.2 向量存储

| 文件 | 格式 | 说明 |
|------|------|------|
| `chroma.sqlite3` | ChromaDB | 实体嵌入向量（使用 BAAI/bge-large-zh-v1.5） |

## 四、数据使用说明

### Q&A 系统数据流

```
用户提问
  │
  ├─→ GraphRAG 本地搜索（ChromaDB 向量检索 + LLM 合成）   [优先]
  │     数据源: entities.parquet + chroma.sqlite3
  │
  ├─→ GraphRAG 全局搜索（社区摘要 + LLM 聚合）            [备选]
  │     数据源: communities_reports.parquet
  │
  ├─→ 关键词检索兜底（SQLite LIKE + LLM 合成）            [降级]
  │     数据源: item.db (500条)
  │
  └─→ LLM 直接回答（无数据源）                            [最终兜底]
```

### 知识图谱可视化

前端 KG 组件查询 `/api/kg/graph`，优先使用 Neo4j（需单独启动），降级到 SQLite 简化图。

## 五、已知局限性

| 问题 | 影响 | 建议 |
|------|------|------|
| item.db 仅 500 条 | 关键词检索覆盖度低 | 扩充至 10,000+ 条 |
| 全为机制银币 | 古钱、铜币等类别无数据 | 补充其他大类数据 |
| 省份识别率 ~50% | 按地域筛选效果差 | 从外部数据源补全 |
| GraphRAG 数据量未知 | 无法评估语义搜索覆盖度 | 输出实体/关系统计 |
| ChromaDB 使用英文 Embedding API | 中文语义检索可能不精确 | 验证嵌入效果 |
