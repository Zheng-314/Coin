# 项目分析与落地计划（2026-02-23）

## 1. 项目定位
- 方向: 钱币/文物知识服务平台（检索 + 问答 + 图谱 + 鉴别 + 估值）
- 当前阶段: 可演示原型，具备比赛展示基础
- 目标阶段: 可部署、可运营、可变现的轻量SaaS

## 2. 当前能力盘点
- 用户系统: 注册、登录、JWT、收藏、聊天记录
- 内容系统: 文物列表、搜索、详情
- 智能能力: 本地/全局问答、联网搜索问答、估值报告
- 图谱能力: 图谱可视化（已改为后端接口供前端消费）
- 视觉能力: 图片上传与预测接口（CPU降级可运行）

## 3. 关键风险
- 安全: 明文密钥、数据库凭据暴露
- 可部署: 前端硬编码地址，环境切换成本高
- 可用性: 问答链路与模型链路存在降级或依赖缺失风险
- 可维护: 数据库建表脚本与实际库结构不一致

## 4. 本次已完成改造
- 移除后端硬编码 OpenAI 密钥，改为环境变量读取
- GraphRAG 初始化增加降级保护，缺少密钥时不致崩溃
- `/api/ask` 修复为真实调用 local/global/web 分支
- 新增 `/api/kg/graph`，前端不再直连 Neo4j
- 前端统一 API 地址配置（`VITE_API_BASE_URL` + `apiUrl`）
- 修复预测页端口不一致问题
- 修复 `favorites` 建表约束为联合唯一（`username, pid`）
- 清理 README 中不安全默认密码提示

## 5. 下一阶段建议（2-4周）
1. 平台化与安全
- 统一 `.env` 模板（开发/生产）
- 关闭 Flask debug，接入 Gunicorn/Uvicorn + Nginx
- 增加请求日志、错误追踪与速率限制

2. 产品化与比赛化
- 做“估值报告”可下载版（含来源与置信说明）
- 增加“问题推荐词”和“搜索来源可信度标签”
- 增加关键指标看板（问答成功率、留存、收藏转化）

3. 商业化
- 免费层: 基础问答 + 基础检索
- 付费层: 深度估值、批量鉴别、收藏跟踪、行情周报

## 6. 部署建议
- 前端: Vercel / Netlify
- 后端: Render / Railway / Fly.io（CPU优先）
- 数据: SQLite 先跑MVP，稳定后迁移 Postgres
- 图数据库: Neo4j AuraDB 或云主机私网部署
